# 📈 Alpha-Talk (알파톡)
> **"투자 아이디어만 말씀하세요. 검증과 최적화는 AI가 합니다."**
> 주린이를 위한 자연어 기반 RAG 퀀트 최적화 플랫폼

---

## 1. 🔍 프로젝트 배경 및 문제 정의 (The "Why")

### 1.1 기존 퀀트 플랫폼의 한계 (Pain Points)
* **높은 진입장벽:** 젠포트(GenPort) 등 기존 서비스는 복잡한 조건식과 금융 용어를 직접 조합해야 하며, 이는 일반 투자자에게 '제2의 코딩'처럼 느껴집니다.
* **논리적 근거 부족:** 왜 이 지표가 중요한지, 현재 시장 상황(금리, 환율 등)과 어떻게 맞물리는지에 대한 전문가적 조언을 얻기 어렵습니다.
* **최적화의 어려움:** 백테스트 결과가 나쁠 때, 어떤 가중치를 어떻게 수정해야 수익률이 개선되는지 스스로 판단하기 어렵습니다.

### 1.2 솔루션: Alpha-Talk의 접근
Alpha-Talk은 **"자연어(NL) 인터페이스"**와 **"검색 증강 생성(RAG)"** 기술을 결합하여, 사용자의 직관적인 아이디어를 정교한 퀀트 전략으로 변환하고 전문가 리포트 기반의 최적화 대안을 제시합니다.

---

## 2. 🎯 SMART 목표 설정

본 프로젝트는 6주간의 집중 개발을 통해 다음 목표를 달성합니다.

* **Specific (명확성):** 사용자의 자연어 입력을 퀀트 엔진용 JSON 파라미터로 90% 이상의 정확도로 변환하고, RAG를 통해 투자 근거를 생성하는 시스템 구축.
* **Measurable (측정 가능성):** AI 최적화 전/후 백테스트 수익률(CAGR) 및 안정성(MDD) 지표를 비교하여 최소 1개 이상의 개선된 전략 제시.
* **Achievable (달성 가능성):** GPT-4o API, Django, 3년 치 KOSPI/KOSDAQ 정적 데이터를 활용하여 6주 내 MVP(Minimum Viable Product) 완성.
* **Relevant (연관성):** 금융 AI 아키텍처 및 퀀트 개발 역량을 증명하며, 실제 투자 의사결정에 도움을 줄 수 있는 실용적 가치 제공.
* **Time-bound (시간 제한):** 2026년 5월(6주 차)까지 통합 테스트 및 최종 배포 완료.

## 3. 📋 요구사항 분석 (Requirements)

### 3.1 기능적 요구사항 (Functional)
1.  **자연어 전략 변환 (NL2JSON):** "성장성 좋은 저평가주 찾아줘"와 같은 입력을 팩터(PER, 매출성장률 등)와 연산자로 변환.
2.  **RAG 지식 베이스:** 증권사 리포트 및 논문 데이터를 기반으로 현재 전략의 타당성 검토 및 보완 아이디어 추출.
3.  **백테스팅 엔진:** 3년 치 일봉/재무 데이터를 바탕으로 전략의 과거 수익률, MDD, 샤프 지수 계산.
4.  **AI 전략 최적화:** 백테스트 병목 지점을 파악하여 지표 가중치(Weight)를 자동으로 조정하고 개선안 제안(Version 관리 포함).
5.  **대화형 리포트:** 텍스트와 차트(수익률 곡선)가 결합된 종합 투자 소견서 출력.

### 3.2 비기능적 요구사항 (Non-functional)
1.  **성능 (Performance):** 고부하 연산인 백테스트와 LLM 호출을 **Celery/Redis**를 통해 비동기로 처리하여 사용자 대기 시간 최소화.
2.  **보안 (Security):** AI가 생성한 수식이 시스템에 직접 영향을 주지 않도록 **White-list 기반 파서** 도입 및 Safe-eval 적용.
3.  **정확도 (Accuracy):** 금융 데이터 특유의 환각(Hallucination) 방지를 위해 **Metadata Filtering** 및 **Strict Grounding** 프롬프트 적용.
4.  **확장성 (Scalability):** 추후 실시간 데이터 스트리밍 및 다중 에이전트(LangGraph) 도입이 용이한 모듈형 아키텍처 설계.

## 기본 프롬프트

너는 20년 차 금융 AI 아키텍트이자 퀀트 개발팀의 테크 리드야.

현재 "주린이를 위한 자연어 기반 RAG 퀀트 최적화 플랫폼"의 전체 시스템 아키텍처와 상세 구현 명세서를 작성해야 해. 

기존 젠포트(GenPort)의 복잡한 UI를 버리고, AI가 사용자의 자연어를 분석해 최적의 투자 전략을 역제안하는 혁신적인 시스템이야.



[프로젝트 핵심 요구사항]

1. 타겟 유저: 주식 용어와 조건식 조합에 약하지만, 논리적인 투자를 원하는 '주린이'.

2. 자연어 전략 생성: 사용자가 대충 "테슬라처럼 크는 주식 찾아줘"라고 입력하면, AI가 이를 정교한 퀀트 팩터(PER, PBR, 매출성장률 등)로 번역.

3. RAG(검색 증강 생성) 지식 베이스: 

   - 기업 분석 리포트, 증권사 시황, 전문가 투자 논문 등을 Vector DB에 적재.

   - LLM이 답변하기 전 무조건 Vector DB를 거쳐 '정확도'와 '전문가적 근거'를 확보할 것.

4. 전략 최적화 및 백테스팅 (핵심 기능):

   - 3년 치 CSV 정적 데이터를 활용해 사용자의 원본 전략을 백테스팅.

   - AI가 더 높은 수익률을 내는 '개선된 전략'을 도출하여 수익률을 비교.

   - "기존 A 지표의 비중을 20% 줄이고, B 지표를 30% 추가하세요"와 같이 구체적인 '수정 비율(Weight)'을 제시할 것.

5. 확장성: 추후 대화형 챗봇(Chatbot)으로 발전할 수 있는 구조 고려.

위 요구사항을 바탕으로 다음 항목들을 상세히 작성해 줘.

1. 시스템 아키텍처 파이프라인 (Data Flow): 사용자 입력부터 최종 AI 리포트 출력까지의 단계별 로직.

2. RAG 정확도 향상 전략: LLM의 환각(Hallucination)을 막고 금융 데이터의 신뢰도를 높이기 위한 청킹(Chunking) 및 프롬프트 엔지니어링 기법.

3. 데이터베이스(ERD) 수정안: 사용자 전략(수식)과 RAG 출처, 최적화 비율을 저장할 핵심 테이블 구조.

4. 백테스팅 엔진과 LLM의 통신 규격 (JSON 예시): AI가 퀀트 엔진에 어떻게 파라미터를 넘기고 결과를 받는지.
